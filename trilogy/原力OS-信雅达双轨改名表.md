# 原力 OS · 信雅达双轨改名表（v0.3）

> **裁决（本表的母法）**：拗口名**不删**——已有势能（书里 55 章、附录 13 卡、外部传播全靠它咬合）一旦改字就断。
> 做法是 **双轨命名**：
> **【专业名】**＝绿皮书正典里的原词，对内、对系统、对老人，一字不改，保留全部势能；
> **【信雅达白话副名】**＝对外、对初学者、对第一次听的人，一句话能听懂、能记住、能复述。
> 副名遵「信雅达」三关：**信**＝不失绿皮书原义（定义取正文，拿不准标 ⚪，绝不硬编）；**达**＝外行听一遍就懂；**雅**＝读起来像中文不像术语缩写。
>
> **本轮边界**：只出**映射表 + 结构修建议**，**不动 OS 绿皮书任何一章正文**。副名是「别名层」，挂在概念旁，不替换正典。

**标记图例**：🟢 已 ground 绿皮书正文定稿　|　⚪ 绿皮书未明写、综合层补的副名（待核）

## 导言 · 为什么要双轨，以及副名怎么取

绿皮书的拗口，**不是写作失误，是钻井深度的副作用**。它逼读者钻到第 1 层（存在性）或第 2 层（第一性），所以满书是 `ʸx`、`IDA`、`OSA`、`模-表-板` 这种「一眼看不懂、看懂了忘不掉」的硬词——对**内核**是优点（信息密度高、咬合紧），对**第一次接触的初学者**是劝退（光符号就吓退一半）。

双轨命名把这对矛盾拆开：

| | 专业名 | 信雅达白话副名 |
|---|---|---|
| **服务对象** | 系统 / 老人 / 正典咬合 | 初学者 / 对外传播 / 第一次听 |
| **取值标准** | 绿皮书原词，零改动 | 信达雅，一句话秒懂 |
| **生命周期** | 永久（改字＝断势能） | 可迭代（副名是营销层，可随渠道调） |
| **存在形态** | 概念正名 | 挂在正名旁的别名 |

副名的工艺三条铁律：
1. **信＞达＞雅**：宁可少一点雅，不可失绿皮书原义。每条副名都对回正文一句话原义。
2. **不抢正名**：副名是「（白话叫……）」，永远跟在专业名后面，不单独出现在正典。
3. **拿不准标 ⚪**：绿皮书没明写的副名（如「硅基三律」是综合层提法），标 ⚪ 待核，不冒充正典。

## §A0 · 核心关窍 · 元AI = 四套东西（产物层 · 先于改名）

> **判断（理论之外，先给「产物」）**：双轨改名解决「概念怎么叫」，但读者更先想知道「**原力OS（元AI）造出来到底长什么样**」。本节立一条**产物层**：原力OS 的核心关窍 = **元AI = 四套东西**，每套绑一个真实可造的系统。理论（信念→图景→工具→身份）是「怎么转」，四套东西是「**转出来的元AI 的解剖**」。

| 套 | 是什么（一句话） | 对应真实系统（生态制品 · 可调） | 在元AI 里的角色 |
|---|---|---|---|
| **一纸文脉** | 你的精神底色——把「你是谁」写成一页可喂给 AI 的上下文 | **原力**：《原力资产》我的说明书 V1-V3 / 龙虾·心智镜AI | **灵魂**（你是谁） |
| **一个大脑** | 专家大脑——把领域知识沉淀成可检索、会成长的知识库 | **OS**：`yuanli-os-company-brain` skill（企业大脑） | **记忆**（你知道什么） |
| **一张地图** | 战略地图——把商业模式 × 关键战役画成可决策的因果图 | **CBM**：cbm-osa atlas / `cascading-causal-cockpit` / `dixin-strategy-cockpit` | **判断**（你怎么决策） |
| **一条链路** | AI 自动化链路——把决策三段 OSA 接成可自动跑的 agent 流水线 | **OSA**：决策三段（§A 第 7 条）→ agent/workflow 自动化 | **手脚**（你怎么执行） |

> **判断（四套 = 元AI 的装配清单）**：文脉=灵魂、大脑=记忆、地图=判断、链路=手脚。四套齐 = 一个**懂你、记得住、会判断、能动手**的元AI。缺文脉→是通用 AI 不是「你的」AI；缺大脑→失忆每次从零；缺地图→瞎干没方向；缺链路→只想不做。这把抽象的「元AI」从一句口号，落成四个能分别动手造的物件——也正是去重缝 3「造物班底归 OS」的具体内容：**资产书引入「元AI 概念」（为你自己造），OS 书把元AI 造成这四套（对外可交付的产能）**。
>
> **与脊骨的咬合**：一纸文脉接脊骨④原力资产（《原力资产》的产出）；一张地图 / 一条链路接脊骨②关窍五步法=信息五层（把碎片炼成全局最优的决策＝OSA）。四套不是新概念，是把已有系统按「元AI 解剖」重新编队。

## §B · 结构修建议（究竟·完备·具体·递归 审计 · 3 条）

> 本节是对绿皮书**结构层**（4 部 / 22 章工具部 / 章内栏目）的审计，**不改正文**，只提改建。每条按「问题（具体引证）→ 改法（可落地）→ 究竟性（为什么这是第一性的修，不是表面缝补）」三段。

### 修 ① · 第三部「工具」22 章臃肿 → 按「对外（项目）/ 对内（人员）」重切

- **问题（具体）**：第三部「工具」是全书最厚的一部（22 章，占全书 40%），README 现按 **A-摄取(7) / B-消化(7) / C-循环(5) / D-节律(3)** 四组划分。但这四组**混了两条正交的代谢轴**：A-摄取讲的是**对外、围绕一个项目周期**的动作（三化改造/解构-建构-增强/模-表-板/验真/复用），B-消化讲的是**对内、围绕人员（你+AI 班底）成长**的动作（班底 1+N/三大脑路由/蒸馏/龙虾/边界）。读者在同一个「工具」大名下，一会儿在做项目、一会儿在养班底，**身份频繁切换**，22 章像一锅烩。
- **改法**：把第三部从「摄取/消化/循环/节律」的代谢拟物分组，**重切为两条显式产线**——
  - **工具 · 对外（项目线）**：吸收原 A-摄取 + C-循环里「怎么对一个项目/客户交付」的章（15-21、29-33）。一句话定位：*一个项目从接到交付的完整动作链*。
  - **工具 · 对内（人员线）**：吸收原 B-消化 + D-节律里「怎么养你自己和 AI 班底」的章（22-28、34-36）。一句话定位：*你和 AI 班底从单兵到自繁殖的成长动作链*。
  - 代谢四词（摄取/消化/循环/节律）保留为**副标题/隐喻**，不再做一级骨架。
- **究竟性**：这不是「换个分类标签」。它对回全书自己的母分类——**对外创造价值 vs 对内维护/增量**（第14章任务族正文已把 7 族归成「对外知识/效率价值」与「对内系统稳定/能力增量」两类）。第三部按「对外项目/对内人员」切，**与第14章的任务族二分同构**，整本书的「对外 vs 对内」轴才贯通。属于结构层向公理对齐的修，不是表面缝补。

### 修 ② · 第二部多章缺「今晚动作」→ 每章补 15 分钟自检

- **问题（具体）**：全书写作纪律是「每章带今晚动作（可落地自检）」，第一部（第03/04/07 章）、第三部、第四部抽样均**有**完整「今晚动作」节。但任务点名的**第二部**里，**第12章信息五层 / 第13章 IDA / 第14章任务族**这三章——经核，第12/13/14 章正文**实际已含「八 · 今晚动作 / 七 · 今晚动作」节**（如第12章「拿一份最近输出逐句标五层」、第13章「三任务按 IDA 写提示词让 AI 数模型钩」、第14章「过去一周任务按 7 族归类统计占比」）。问题不在「缺」，而在**这三章的今晚动作偏重，单次耗时远超 15 分钟**（逐句标注整篇/写 3 个完整 IDA 提示词/归类 20-50 个任务），对初学者门槛高、易弃做。
- **改法**：保留原「今晚动作」为**进阶版**，每章**增挂一个「15 分钟轻量版」**自检入口（降门槛、保留即时反馈）——
  - 第12章：轻量版＝只挑**最近一份输出里最重要的 1 句话**，判它在第几层、能不能推到层 4（而非逐句标全篇）。
  - 第13章：轻量版＝只把**下一个任务**按 I/D/A 各写 1 句，自查有没有「让 AI 做决策」的颠倒（而非写 3 个完整提示词）。
  - 第14章：轻量版＝只给**今天接的下一个任务**贴一个族标签 + 判断是否混合族（而非归类整周）。
- **究竟性**：与第一法印「人类友好」直接同构——今晚动作过重＝违反「不打扰/保护启动者」，初学者第一次就被劝退，递归进化的「第一次留痕」根本发生不了。轻量版是把「每章可落地」这条写作宪法，从「资深者能做」修到「初学者也能做」，守的是 ʸx 嵌套链「启动者必须先动起来」的根。

### 修 ③ · 第三/四部每个概念旁加「对应左腿第 X 关」标签

- **问题（具体）**：原力体系是**左腿（创业/战略·借势合力四关）× 右腿（OS·工作系统）** 双腿并行（见 `原力三部曲-基石与关键概念-全谱` §0 母层）。但 OS 绿皮书（右腿）第三/四部的工具与身份概念，**与左腿四关**（`gates.jsx` verbatim：**原力觉醒 / 品类独创 / 模式升维 / 壁垒锁定**）**没有显式咬合标记**。读者学完 OS 的「三化改造」「商业模式九模块」，不知道它对应创业旅程的哪一关，两腿各走各的、不交叉受力。
- **改法**：第三/四部每个概念旁加一枚 **「↔ 左腿第 X 关」** 徽章（只加标签，不改正文），初版映射建议（拿不准标 ⚪ 待核）——
  - **↔ 第一关 原力借势**（gates.jsx 现「原力觉醒」，改名中）：三非心智筛 / 龙虾即心智外显 / S 路径（认领「我是哪个版本号」）。
  - **↔ 第二关 品类独创**：三化改造（把隐知识做成独有品类资产）/ 信息五层（洞察密度＝差异化内核）。
  - **↔ 第三关 模式升维**：班底 1+N / 蒸馏 / 模-表-板（从单次交付升维到可复用系统）/ ⚪ 商业模式九模块（第46章）。
  - **↔ 第四关 壁垒锁定**：双 OS 架构（治理护城河）/ 单一真源与自持知识库（第32章）/ ⚪ 虚实出入四壁垒（第46章）。
- **究竟性**：这是把白皮书「两条腿 + 一层哲学贯穿（三投影）」的母结构（同一 ʸx 照在「怎么变现/怎么变强」两面墙）**在概念层落地**。不加这枚徽章，左右腿在字典层是断的；加上之后，读者每学一个 OS 工具都知道它在创业旅程的承重位置——属于结构层把「双腿同源」从口号修成可导航的咬合，是第一性的修。
